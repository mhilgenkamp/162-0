# 162-0 — Project Context

**Live URL:** https://162-0.vercel.app (or check Vercel dashboard)
**GitHub:** https://github.com/mhilgenkamp/162-0

Inspired by [82-0.com](https://www.82-0.com/) — a basketball draft game. This is the baseball equivalent.

---

## What it is

A browser game where you draft a baseball roster one player at a time. Each round, a **team + decade** combination is spun and you pick any player from that pool. After 15 picks your roster is revealed with a projected win total. The goal is to build a team good enough to go 162-0.

---

## Tech Stack

- **Next.js 14** (App Router, `"use client"` components)
- **TypeScript + Tailwind CSS**
- **No external API** — all player data is a local JSON file (`lib/players.json`)
- Deployed on **Vercel**

## Game Mechanics

### Draft flow
1. A slot machine spins a random **team + decade** (e.g. "New York Yankees · 1990s")
2. All eligible players from that exact team+decade are shown in a scrollable, filterable, searchable list
3. Player picks one — that player fills their natural roster position
4. Repeat for 15 rounds

### Roster slots (15 total)
- **9 position players:** C, 1B, 2B, 3B, SS, LF, CF, RF, DH
- **6 pitchers:** SP × 3, RP × 3
- **DH is a utility slot** — any hitter can fill it if their natural position is already taken. The player list shows `→ DH` on the position badge when this routing applies.

### Skips
- 1 **Team skip** (re-spin keeping same decade)
- 1 **Decade skip** (re-spin keeping same team)

### Auto-reroll
If a team+decade spin yields zero eligible players (e.g. a franchise that didn't exist in that decade), the game silently re-spins until it finds a valid pool. Cap of 50 attempts.

### Game modes
- **Classic** — all stats shown during selection
- **Baseball IQ** — stats hidden, pick on name recognition alone

### Win projection (hidden during draft, revealed at end)
```
Projected Wins = 48 + Σ(WAR of all 15 players)
```
- `48` = replacement-level team baseline (standard baseball analytics)
- Each player's WAR = their **best 3-year average WAR** from within that franchise+decade
- To reach 162-0 you need ~114 total WAR — requires elite picks at every slot
- The projected record is intentionally hidden during the draft for a surprise reveal

## Player Data

### Source
- **Lahman Database** (SQLite, `scripts/data/lahman.sqlite`) — season stats for every MLB player since 1871
- **Baseball Reference WAR** (`scripts/data/war_daily_bat.txt`, `war_daily_pitch.txt`) — season bWAR for every player

### Build script
`scripts/buildPlayers.ts` — run with `npx tsx scripts/buildPlayers.ts`

Outputs `lib/players.json` (~1.3MB, ~4,459 records). The data files in `scripts/data/` are gitignored (too large).

**To re-download data files:**
```bash
# Lahman SQLite
curl -L "https://github.com/jknecht/baseball-archive-sqlite/releases/download/2022/lahman_1871-2022.sqlite" \
  -o scripts/data/lahman.sqlite

# bWAR
curl https://www.baseball-reference.com/data/war_daily_bat.txt   -o scripts/data/war_daily_bat.txt
curl https://www.baseball-reference.com/data/war_daily_pitch.txt -o scripts/data/war_daily_pitch.txt

npx tsx scripts/buildPlayers.ts
```

### Franchise handling
Historical team relocations are unified under the modern franchise name via Lahman's `franchID`:
- Brooklyn Dodgers → `LAD` → "Los Angeles Dodgers"
- New York Giants → `SFG` → "San Francisco Giants"
- Philadelphia/Kansas City A's → `OAK` → "Oakland Athletics"
- Montreal Expos → `WSN` → "Washington Nationals"
- etc.

### Player ID format
`{lahmanPlayerID}-{FranchiseName}-{decade}-{bat|pit}`
e.g. `ohtansh01-LosAngelesAngels-2020s-bat` and `ohtansh01-LosAngelesAngels-2020s-pit`
(Ohtani has separate entries as DH and SP)

### Filters
- Hitters: minimum 100 PA/season in the 3-year window
- Pitchers: minimum 15 IP/season
- Minimum 3-year avg WAR ≥ 1.5 to appear in dataset

| File | Purpose |
|------|---------|
| `lib/types.ts` | All TypeScript types (Player, RosterSlot, GameState, etc.) |

| `scripts/buildPlayers.ts` | Data pipeline: Lahman + bWAR → players.json |

## Known Issues / Future Work

- **DH players from pre-DH era** — DH didn't exist in the NL until 2022; some decade/position combos will have no DH options
- **OF position splitting** — Lahman's generic "OF" is mapped to LF via `FieldingOFsplit`, but occasionally a player gets LF when they were actually a CF/RF
- **WAR threshold** — 1.5 is the current floor; lowering it adds more players (thinner teams like the 2000s Orioles get better pools); raising it creates more scarcity
- **No mobile layout** — currently desktop-optimized; the player list needs responsive work for small screens
- **Complete screen** — the end screen shows projected record and roster but no per-player WAR breakdown; a reveal animation and WAR-per-slot display would improve it
- **Score sharing** — no way to share your final roster/record with friends yet
- **Slot machine animation** — currently just a text blur; could be improved with a proper reel animation
- **`PlayerCard.tsx` is unused** — was replaced by `PlayerList.tsx`; can be deleted
